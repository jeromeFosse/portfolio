Partie 1: LES COMPOSANTS D'UN ORDINATEUR

  • Boîtiers
  • Boites d’alimentation :
  • Carte Mère : élément centrale, principale carte de circuits imprimés.
  • Socket : support processeur
  • Chipsets : gère les flux entre les composants (▪ Pont nord : processeur, mémoire vive, carte graphique ▪ Pont sud : périphériques entrés et sorties)
  • Port PCI : connecte carte graphique, son et réseau
  • Port ATA / SATA : connecte disques durs ou lecteurs optiques
  • Port PS2, parallèle et série : connecte clavier, souris, imprimante, …
  • Réseau : RJ45
  • Multimédia : VGA, DVI, HDMI
  • Processeur : Cerveau de l’ordinateur, lieu des calculs effectués par l’ordinateur.
  • Système de refroidissement
  • ROM : Mémoire morte, stockage de données indispensable au démarrage.
  • RAM : Mémoire vive, stockage temporaire des données et programmes que le programme est en train d’utilisé. Mémoire volatile, le contenu s’efface.
  • Périphériques d’entrée : Souris, clavier, scanner, appareil photo, …
  • Périphériques de sortie : moniteurs, imprimantes, télécopieurs, casques, …
  • Entrée et sortie : modem, disque dur, carte mémoire, USB.

Partie 2: LE PROTOCOLE HTTP

  • Web évolutif : web 1.0 (site statique, informatif), web 2.0 (réseaux sociaux, échange), web 3.0 (web sémantique, recueille information utilisateur) et web 4.0 (web intelligent)
  • URL : Uniform Resource Locator
  • Http : protocole qui permet le transfert de fichiers grâce à une chaine de caractères URL entre un navigateur et un serveur Web (requête/réponse)
  • Méthode GET : Affichage
  • Méthode POST : Envoie
  • Sites statiques : envoie de fichiers hébergés « tels quels » vers le navigateur
  • Sites dynamiques : il met à jour les fichiers hébergés avant de les envoyer. Le résultat dépend des demandes effectuées.

Partie 3: LES SYSTÈMES D'EXPLOITATION

  • Code source : séquence d’instructions compréhensible par l’homme transformée en langage machine
  • Open source : disponible à tous
  • Android : open source développée par google
  • iOS : fermé par Apple
  • Linux 
  • HTC Dream : système d’exploitation Android utilisé sur différents appareils
  • Unix : système d’exploitation non-propriétaire, c’est la base de tous les systèmes d’exploitation connu (linux, android, OS X, iOS)
  • Interpréteur de commandes : interprète les commandes et les transmet au système d’exploitation

Partie 4: LA VIRTUALISATION

  • Faire fonctionner plusieurs systèmes d’exploitation et/ou applications sur une machine
  • Virtualisation des postes de travail : administrateur central, réseaux informatiques
  • Virtualisation des serveurs : conçus pour traiter un volume élevé de tâches spécifiques, cela permet aux ordinateurs d’effectuer diverses autres tâches. La virtualisation permet optimiser l’exécution
  • Hyperviseurs : Permet de faire cohabiter simultanément plusieurs systèmes d’exploitation, il est entre le matériel et les logiciels
  • Hyperviseur type 1 : directement sur la machine (source)
  • Hyperviseur type 2 : logiciel qui s’exécute à l’intérieur d’un système d’exploitation

Partie 5: INTRODUCTION LINUX

Arborescence et Répertoires :

  • / : répertoire racine
  • /bin : utilitaire système standard, répertoire utilisateur, groupe,
  • /sbin : Exécutable système essentiels, répertoire système d’exploitation
  • /boot : programme d’amorçage
  • /dev : fichier pilotes de périphériques
  • /etc : configuration système machine
  • /home : répertoires personnels utilisateur
  • /lib : bibliothèques partagées
  • /media : point de montage des périphériques cd-rom, appareil photo, USB
  • /mnt : points de montage temporaires
  • /proc et /sys : Répertoire virtuel pour les informations système
  • /root : répertoire personnelle de l’utilisateur root
  • /tmp : stockage de données temporaires
  • /usr : données partageables et en lecture seule hiérarchie secondaire
  • /var : taille de fichiers imprévisible, données variables
  • /srv : données pour les services fournis par le système
  • /opt : répertoire d’applications, logiciel

Les chemins :

  • Path : emplacement unique vers un fichier ou un dossier dans un système de fichier
  • Absolu : chemin à partir du répertoire racine
  • Relatif : chemin à partir du répertoire de travail actuel

Partie 6: LES COMMANDES AVEC LINUX

Générale :

  • pwd : affiche le chemin d’accès
  • ls : affiche la liste des dossiers ou on se trouve
  • ls –l : donne les détails des dossiers/ documents
  • cd nomDossier/ : entre dans le dossier
  • cd .. : retourne au dossier précédent
  • ctrl l : vide la fenêtre
  • su = passe en mode root (# : quitte le mode)
  • ctrl shift t : ouvre un 2ème terminal

Création de document / répertoire :

  • nano : crée un fichier texte ( ctrl o : enregistre, ctrl x : quitte le document)
  • mkdir nomDossier : crée un dossier
  • touch nomFichier : crée un fichier
  • touch -t 10251600 nomFichier : Change la date du fichier

Visualisation des fichiers :

  • more nomFichier : affiche le contenu du fichier dans le terminal
  • less nomFichier : affiche le contenu dans le fichier(q : sortir)
  • cat : visualiser des fichiers court. Pas de défilement

Copier / Déplacer :

  • cp nomFichier nouveauNomfichier : copie et nomme le nouveau fichier
  • mv nomFichier nouveauNomFichier : renomme le document
  • mv -t nomDossier/ NomFichier nomFichier: déplace plusieurs fichiers dans un dossier
  • mv nomFichier nomDossier/ : Déplace un fichier dans un dossier
  • -f : permet d’écraser si même fichier et même nom

Suppression :

  • rm nomFichier : supprime le fichier
  • rm -rf = force la suppression du répertoire
  • rmdir : supprime un répertoire
  • rm -f nomFichier* : supprime les fichiers commençant par

Documentation linux :

  • ls --help : affiche un court message d’aide
  • man ls : page de manuel
  • man -k trace : effectue une recherche par mot clés
  • info -ls : structure plus robuste que man

Visualiser les utilisateurs et les groupes :

  • etc/ : dossier utilisateur
  • /etc/passwd : liste des utilisateurs
  • /etc/shadow : fichier mots de passe chiffrés
  • /etc/NomGroup : liste des groupes

Création de Groupe et Utilisateur :

  • useradd / groupadd : ajout
  • usermod / groupmod : modification
  • userdel / groupdel : suppression

Exemple (ajout dans root) :

  • sudo groupadd groupe1 : creation du groupe1
  • sudo useradd – m Alice : création de l’utilisateur Alice
  • ls -l /home/ : donne les informations des utilisateurs avec date

Téléchargement d’application sur Linux :

  • apt-get update : vérifie la mise à jour
  • ping URL : test la connexion
  • apt install wget : installe le package

Installation d’application :

  • Aller sur le site de l’application
  • apt update : mettre à jour
  • apt install Name : permet d’installer (ex : snapd)
  • snap install Name : installe l’application (ex : whatsdesk)
  • Sortir de root et mettre nom (ex : whatsdesk)

Partie 7: LES GESTIONNAIRES DE VERSION

  • Programme qui permet de conserver un historique des modifications et des versions des fichiers
  • 3 fonctionnalités : ▪ Revenir à une version précédente du code si problème ▪ Suivre l’évolution ▪ Travail à plusieurs

Git ou GitHub :

  • Git est un gestionnaire de version : dépôt local (sur la machine)
  • GitHub est un service en ligne : dépôt distant

Partie 8: LES ÉTAPES ET CREATION DÉPOT GIT

Installation de Git :

  • apt-get update && apt-get upgrade : vérifie l’installation et si il y a des mises à jours
  • apt get install git: met en place git

Configuration de l’identité :

  • git config --global user.name “” : configuration de username
  • git config --global user.email “” : configuration du courriel
  • git config --list: Vérifier les paramètres

Cloner le projet de GitHub à l’ordinateur :

  • git clone http….web.git : adresse prise dans le projet GitHub

Pour créer une nouvelle version et l’envoyer sur GitHub :

  • git add nomFichier : index le fichier
  • git commit -m « texte » : explique l’étape faite (résumé)
  • git push : envoi le commit sur le dépôt distant
  • git log : permet de voir les commits réalisés

Création de branche :

  • git checkout NomBranche : basculement sur la branche.
  • git branch : permet de montrer sur qu’elle branche on se trouve
  • git checkout -b NomNouvelBranch : créer une nouvelle branche et bascule sur cette branche
  • git merge NomFichier : fusion entre 2 branches

Partie 9: LES LANGAGES DE PROGRAMMATION

  • Langage populaire : Java, python, c# et c/++, php, javascript
  • Éditeurs de texte : Atom, bracket, visual studio code
  • IDE ou EDI: Environnement de Développement Intégré, c’est un ensemble d’outils qui permet d’augmenter la productivité.

Partie 10: L'ENVIRONNEMENT DE DÉVELOPPEMENT ET DE TEST

  • Architecture Client – Serveur : c’est une requête faite d’un client à un navigateur, c’est la communication entre le client et le serveur
  • PhpMyAdmin : interface de gestion de base de données MySql sur un serveur PHP. Elle permet de faire des requêtes comme les créations de table de données, insertions, mises à jours, suppression et modifications de structure.
  • MySQL : stock les données dans des tables séparées. Les tables sont reliées par des relations définies, qui rendent possible la combinaison de données.
  • Apache : logiciel de serveur web gratuit et open-source. Il interprète les requêtes http arrivant sur le port et fourni une réponse.

Partie 11: INTRODUCTION À LA CYBERSÉCURITÉ

  • Logiciel publicitaire : publicités envahissantes et indésirables
  • Logiciel espion : s’installe furtivement sur un ordinateur pour recueillir des informations sur l’utilisateur
  • Logiciel Rootkit : il est furtif qui permet aux pirates d’accéder à votre ordinateur à votre insu. Le pirate a un contrôle absolu sur l’appareil
  • logiciel Ransomware : est un logiciel qui empêche l’utilisateur d’accéder à leur système ou à leurs fichiers et exige une rançon.
  • Un Ver : il s’exécute, se reproduit et se propage dans l’ordinateur.
